== Crystals


The topic of the semester project is crystals. Your task is to write a module in OpenSCAD that can generate crystal models based on input data (and a module that generates random crystals), suitably prepare one of your assigned crystals for printing, print it, and perform any necessary additional adjustments.

WARNING: We welcome any questions about issues with this assignment via an
https://github.com/3DprintFIT/B252CW-Assignment/issues[issue] in this repository.

=== Crystal generator


Write an OpenSCAD module `crystal` adhering to the prescribed interface, which displays/creates a crystal based on input data.


----
module crystal(thorns=[]) { ... }
----

The vector (array) `thorns` contains definitions of the individual outgrowths (thorns) of the crystal. A single thorn is defined by its rotation, length, radius, and number of segments.


----
thorns_abstract = [[[rotx, roty, rotz], height, circumr, seg], ...];
thorns_example = [[[0,0,0], 30, 3, 4], [[90,0,0], 25.5, 2.65, 7]];
----

Each thorn has a shape similar to the one in this image:


image::rect10491.png[height=300]

The center of the bottom side lies at the origin of the coordinate system. Before rotation, the thorn points straight up (along the positive Z axis). The total length of the thorn is `height`, the kink is the cube root of `height` away from the top. The thorn has the profile of a regular ("equilateral") _n_-gon, where _n_ is `seg`. The circle circumscribed around this _n_-gon has a radius of `circumr` at the location of the kink, and one third of that radius at the bottom of the thorn. The thorn is then rotated by the vector `+[rotx, roty, rotz]+`.

A crystal is simply a mixture of these thorns.

For the two-thorn example given above, the result looks roughly like this:


image::2example.png[height=300]

The next task is a module that generates random crystals. It does not receive specific data as input, but only their bounds (closed intervals) and the number of thorns.


----
module random_crystal(nthorns=350, rot=[-90,100], height=[20,40], circumr=[2,4.5], seg=[3,10]) { ... }
----

* `nthorns` is the number of thorns
* the other arguments are two-element vectors specifying the minimum and maximum values for the thorn properties of the same name (`rot` are the bounds for `rotx`, `roty`, and `rotz`)

For generating random numbers, you can use the function https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Mathematical_Functions#rands[rands()]. Keep in mind that `seg` (unlike the other random values) is an integer. Each individual integer, including the boundary values, should have the same chance (uniform distribution); be careful, failing to satisfy this is penalized in the grading.

For the default values, a crystal can look various ways (it is random), for example like this:

image::350example.png[height=300]

NOTE: It is not necessary to do this part of the semester project, you can use a ready-made STL file.

=== Pre-processing


Choose any (in your opinion the simplest) one of your three crystals and prepare it for printing. You can do practically anything with it (repair, rotate, slice, add support structures), but the dimensions and shape of the crystal must be preserved during printing. The output is one or more STL files ready for slicing and a very brief description of *what* you did and *why* (not necessarily in writing, but at submission time you must be able to explain the procedure, even several weeks after performing your changes).


=== Slicing


Slice the print STLs you prepared using any program, with appropriate settings for our Voron printers. For OrcaSlicer, start from the profiles used in the labs. We do not have profiles for any other programs, but you may create your own. The output is a 3mf file from Orca or an exported slicing profile from another program, and one or more GCODE files. The individual parts can be printed all at once (if they fit on the print bed and you find it appropriate) or sequentially, or a combination of both.


=== Printing and post-processing


During the credit sessions listed in KOS, taking place in the exam period, you will print crystals in the lab on the Voron printers from PETG material using the GCODE files you prepared. After printing, the print must be properly processed - removing supports, gluing, etc. The resulting crystal should look as similar as possible to the assigned model. There is a maximum of 5 hours per session in total (printing + post-processing).

In case of an absolute failure during printing, it is possible to repeat the print with new GCODE, but only once. In the case of a technical problem on our side, it is of course not considered a wasted attempt.


== Submission, grading

Submission is on GitHub: https://classroom.github.com/a/PzGZON0y

It must be submitted by the time of your credit session.
The dates will be announced at the end of the semester.

All written descriptions should go directly into the README (or README.md, etc.) in
the repository. **This time, do not expect any automatic issue.**

Submit in the repository:

- a scad file with the `crystal` and `random_crystal` modules
- a scad file **calling** the `crystal` module with your selected data (without the declaration/definition of the `crystal` module)
- an STL file with your selected crystal from ownCloud
- All print STL files
- All print GCODE files
- a 3mf file from OrcaSlicer with all slicing settings and/or a profile for another slicing program you used
- Any other necessary files

Grading according to the following table:

|===
| **Part** | **points** | **note**
| **Modules for OpenSCAD:** | **10** |
| The `crystal` module works according to the assignment | 3 | mandatory within the part
| The `random_crystal` module works according to the assignment | 4 | mandatory within the part
| The source code is suitably structured and commented | 3 |
| **Print preparation:** | **10** |
| Suitably prepared print STLs | 5 | mandatory within the part
| The mesh in all print STLs is in order | 5 | for errors not affecting the print, only 2.5 points are deducted
| **Slicing:** | **10** |
| Supports | 5 | not needed (5 pts), suitable use* (2.5 pts), unnecessary use (0 pts)
| Suitable settings of print parameters | 5 | perimeters, infill, layer height...
| **Printing:** | **10** |
| It is a print of the model according to the assignment, the print is processed (e.g. without supports, glued, etc.) | 4 | mandatory within the part
| The print does not contain defects clearly caused by unsuitable preparation of the model | 3 |
| The print does not contain defects clearly caused by unsuitable preparation of the printer (preparation of the print bed, unsuitable temperatures) | 3|

* Points are only deducted for supports generated during slicing. Because we are in the slicing part.

The grading is divided into 4 sub-parts. *Mandatory within the part*
means that without fulfilling this task, the student will not receive any
points for the given part. In the case of a re-print, the point values in
the other sub-parts are no longer corrected. So if, for example, you fail at slicing,
you get zero points from it and (quite logically) the print will fatally fail too,
you can get points for the printing on a make-up date, but you will not get any
points for the slicing anymore.
